Нужно написать два варианта fork-бомбы.

	1. Безопасную, содержащую цикл:

		while(1) { fork(); }
		
		с которой можно справится, нажав ^C (ctrl+C) в консоли.

	2. Опасную, с которой уже справится не так просто, через установку
		группы для от-fork-нутой группы нитей примерно по следующей
		схеме:

		p = fork();
		if(p)
		{
			...
		}
		else
		{
			setpgid(...);
			print(getpgid(0)); // (*)
			while(1) { fork(); }
		}

		В этом случае нужно.
			1. Убедиться (запустив top -HU username), что ^C не
				прервёт выполнение fork-бомбы.

			2. При помощи информации, выдаваемой в строке (*)
				прервите группу fork-бомбы при помощи команды
				kill через 10 секунд.

				Сделать это можно по следующей схеме:

				./fb | (
					read fbpid;
					sleep 10s;
					kill -INT -$fbpid ) # (+)

				Только вместо | следует использовать
				именованный канал связи, который можно создать
				утилитой mkfifo.

				Потому что иначе все процессы команды (+) будут
				прерываться при помощи ^C (и sleep, и kill).

Решением задания должны быть две программы на C. И скрипт, запускающий и
прерывающий опасную fork-бомбу.

НЕ ЗАБЫВАЙТЕ ПРО ОБРАБОТКУ ОШИБОК. Если не удастся вывести в stdout
номер группы в опасной бомбе, то она станет действительно опасной, и от неё
систему, скорее всего, получится спасти только при помощи перезагрузки.
